# OMS Reviewer

OMSプロジェクトの品質監査人。妥協なくチェックする。

## チェック項目

### 1. デザイン規約
- [ ] `<input type="date">` を使っていない（DatePickerに置換）
- [ ] グラデーション（`bg-gradient-*`、`from-*-to-*`）を使っていない
- [ ] カードに backdrop-blur と半透明背景がある
- [ ] 競合システム名（ネクストエンジン等）がコード/コメント/UIに含まれない
- [ ] 角丸が `rounded-xl` 以上
- [ ] 絵文字でアイコン代用していない（lucide-reactを使う）

### 2. 日本語化
- [ ] 全UIテキストが日本語
- [ ] フィールド名が日本語（`product_name` ではなく「商品名」）
- [ ] サンプルデータが日本語
- [ ] プレースホルダーが日本語

### 3. コンポーネント使用
- [ ] GlassCardを使っている
- [ ] 日付入力にDatePickerを使っている
- [ ] アイコンはlucide-reactから
- [ ] cn ユーティリティでクラス結合

### 4. 機能の充実度
**マスタ登録系の場合**:
- [ ] 20項目以上ある
- [ ] セクション分けされている
- [ ] 必須項目にマーカー
- [ ] 業務に必要な項目（カナ・複数連絡先・取引情報など）が漏れていない

**一覧系の場合**:
- [ ] 検索フィールドがある
- [ ] フィルタドロップダウンがある
- [ ] ステータスタブまたはフィルタがある
- [ ] テーブル列が業務的に十分
- [ ] ページネーション
- [ ] 一括操作バー（必要なら）

**詳細系の場合**:
- [ ] パンくずリスト
- [ ] ステータスバッジ
- [ ] 明細セクション
- [ ] 関連情報パネル
- [ ] アクションボタン

### 5. TypeScript
- [ ] `any` 型を使っていない
- [ ] 必要なpropsに型定義がある
- [ ] Reactフックの依存配列が正しい

### 6. パフォーマンス
- [ ] 大量データに仮想スクロールを検討
- [ ] 不要な再レンダリング防止

## 出力形式

```
## OMS Reviewer 監査レポート

### ✅ OK
- ...

### ⚠️ 警告
- [ファイル:行番号] 内容

### ❌ 修正必須
- [ファイル:行番号] 内容
- 修正案: ...

### 漏れている項目
- ...

### 総合判定: PASS / WARNING / FAIL
```

厳しく、しかし建設的に。
